# Learner Engine & Adaptive Memory

## Overview
The **Learner Engine** acts as the system's "memory," responsible for tracking the student's progress, mastery levels, and optimal review schedules. It transforms the ITS from a stateless chatbot into a long-term educational companion.

## 🧠 Cognitive Architectures Implemented

### 1. BKT (Bayesian Knowledge Tracing)
BKT is a mathematical model used to estimate the probability that a student has mastered a specific skill based on their sequence of correct and incorrect attempts.

**How it works:**
Every time a student attempts a question (e.g., "Python Loops"), the system updates four probabilities:
*   P(L): Probability the skill is already known.
*   P(T): Probability of learning the skill during the step.
*   P(G): Probability of guessing correctly (Guess).
*   P(S): Probability of making a mistake despite knowing the skill (Slip).

**Code Reference:** `src/core/adaptive/bkt.py`

### 2. Spaced Repetition (Ebbinghaus Forgetting Curve)
To prevent students from forgetting what they've learned, we use a modified **SM-2 Algorithm** (similar to Anki/SuperMemo).

**The Logic:**
*   If a student answers correctly, the interval for the next review expands (e.g., 1 day -> 6 days -> 15 days).
*   If they answer incorrectly, the interval resets to 1 day, forcing immediate review.
*   The system calculates a `due_at` timestamp for every skill.

**Code Reference:** `src/core/adaptive/srs.py`

## 🔄 Data Flow (The "Update Loop")

1.  **Student Attempt:** Student submits an answer to `/attempt`.
2.  **Assessment:** The Assessment Engine grades it (0.0 to 1.0).
3.  **Learner Update:** The Learner Engine receives the score and triggers:
    *   **BKT Update:** Calculates new mastery probability (P_new).
    *   **SRS Update:** Calculates next review date (`next_review`).
    *   **Global State:** Updates overall course `readiness_score`.
4.  **Database:** All new values are saved to `learner_skills` and `learner_schedules` tables.
